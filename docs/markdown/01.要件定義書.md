# 要件定義書

## エンジニア向けリソース・プロジェクト管理SaaS

------------------------------------------------------------------------

# 1. システム概要

## 1.1 システム名称（仮）

ResourceFlow（仮）

## 1.2 目的

エンジニア組織において以下の課題を解決する。

-   誰がどのプロジェクトにアサインされているか分かりにくい
-   各メンバーの稼働率が可視化されていない
-   タスクと人員配置が分離して管理されている
-   プロジェクトごとの進捗が俯瞰しづらい

本システムでは

-   メンバー
-   プロジェクト
-   タスク
-   アサイン（稼働率）

を統合管理することで、エンジニア組織のリソース状況を可視化する。
