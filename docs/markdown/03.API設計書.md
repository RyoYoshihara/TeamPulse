# API設計書
## エンジニア向けリソース・プロジェクト管理SaaS

---

# 1. 前提

- バックエンドは FastAPI を採用する
- API形式は REST
- 認証は JWT または HttpOnly Cookie を想定
- すべての業務データは組織単位で分離する
- APIパスのベースは `/api/v1` とする

---

# 2. 共通仕様

## 2.1 レスポンス形式

### 成功時

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

### エラー時

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容に誤りがあります"
  }
}
```

---

## 2.2 主なHTTPステータス

|ステータス|用途|
|---|---|
|200|取得成功・更新成功|
|201|作成成功|
|400|リクエスト不正|
|401|未認証|
|403|権限不足|
|404|対象なし|
|409|競合|
|422|バリデーションエラー|
|500|サーバーエラー|

---

## 2.3 エラーコード例

|コード|説明|
|---|---|
|UNAUTHORIZED|認証失敗|
|FORBIDDEN|権限不足|
|VALIDATION_ERROR|入力不正|
|NOT_FOUND|対象データなし|
|DUPLICATE_RESOURCE|重複登録|
|ASSIGNMENT_OVER_CAPACITY|配属率超過|
|INTERNAL_SERVER_ERROR|予期しないエラー|

# 3. 認証API

## 3.1 ログイン
- **POST** `/api/v1/auth/login`

### Request
```json
{
  "email": "admin@example.com",
  "password": "password"
}
```

### Response
```json
{
  "success": true,
  "data": {
    "access_token": "jwt-token",
    "token_type": "bearer",
    "user": {
      "id": "uuid",
      "name": "Admin User",
      "email": "admin@example.com",
      "role": "admin"
    }
  }
}
```

---

## 3.2 ログアウト
- **POST** `/api/v1/auth/logout`

---

## 3.3 自分情報取得
- **GET** `/api/v1/auth/me`

---

# 8. アサイン管理API

### Query
- `project_id`
- `member_id`
- `start_date`
- `end_date`

---

## 8.2 アサイン作成
- **POST** `/api/v1/project-assignments`

### Request
```json
{
  "project_id": "uuid",
  "member_id": "uuid",
  "allocation_rate": 50,
  "start_date": "2026-04-01",
  "end_date": "2026-06-30",
  "role": "backend",
  "is_primary": true
}
```

### バリデーション
- `allocation_rate` は 0〜100
- 同時期の総配属率超過時は警告またはエラー

---

## 8.3 アサイン更新
- **PATCH** `/api/v1/project-assignments/{assignment_id}`

---

## 8.4 アサイン削除

# 13. 権限設計（初期案）

## admin
- 全機能操作可
- ユーザー管理可
- 組織設定可

## manager
- メンバー、プロジェクト、タスク、アサインの管理可
- ユーザー管理は制限付き

## member
- 自分に関係するタスク閲覧・更新
- 自分の工数登録
- コメント投稿
